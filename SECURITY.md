# Security Policy

## Supported versions

| Version                       | Supported |
| ----------------------------- | --------- |
| Latest public release         | Yes       |
| Older releases                | Limited   |
| Unreleased development builds | No        |

Users should always install the latest official version of Vision Guard PC.

## Reporting a vulnerability

Do not publish security vulnerabilities in a public GitHub Issue.

Report vulnerabilities privately through the contact options available on the GitHub profile of:

**Maurício Morais Zage**
GitHub: [MauricioMoraisZage](https://github.com/MauricioMoraisZage)

Include:

* Vision Guard PC version;
* Windows version;
* clear description of the vulnerability;
* steps to reproduce it;
* possible impact;
* screenshots or logs, when appropriate;
* suggested mitigation, when available.

Do not include passwords, personal data, private files, authentication tokens, or other sensitive information.

## Responsible disclosure

Please allow reasonable time for the vulnerability to be investigated and corrected before publishing technical details.

Security reports made in good faith will be treated respectfully.

## Official downloads

Only download Vision Guard PC from official sources:

* the official GitHub Releases page;
* Microsoft Store, when available;
* other channels explicitly announced by the author.

The author is not responsible for modified copies or installers obtained from unofficial websites.

## Scope

Security reports may include:

* unauthorised access to local application data;
* insecure file permissions;
* arbitrary code execution;
* privilege escalation;
* unsafe system-command execution;
* update or installer vulnerabilities;
* bypasses of application protections;
* exposure of sensitive local information.

General bugs, interface problems, feature requests, and wellness recommendations should be submitted through normal GitHub Issues.

## Privacy

Vision Guard PC currently stores application information locally and does not intentionally transmit usage history to external servers.

See [PRIVACY.md](PRIVACY.md) for additional information.
